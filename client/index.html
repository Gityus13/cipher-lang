ⓘ HOW IT WORKS
// LAYER 1 — REAL SECURITY
① Plaintext → AES-256-GCM (random 96-bit IV per message)
② GCM produces ciphertext + 128-bit authentication tag
③ Key lives server-side only — never sent to browser
// LAYER 2 — SECRET LANGUAGE
④ Every byte → unique symbol (Δ/Ω/Λ/Σ/Ψ/Θ/Ξ/Γ + hex)
⑤ Random noise tokens injected → identical input ≠ identical output
⑥ Wrapped: ⟦KID<id>::iv::authtag::ciphertext+noise⟧
// VISIBILITY SYSTEM
⑦ Public = stored + visible in Vault to all logged-in users
⑧ Private = stored + only you can see it in the Vault
⑨ Both require this server's key to decode — no offline decoding
